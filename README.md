# [電子パーツ在庫ビューア](https://46nori.github.io/ePartsDB/)

個人所有の電子部品のデータベースとビューアです。

以下のような状況がよく起こるので、外出先からスマホでも簡単に確認できるようにしてみました。

- 手持ちの部品で設計したいけど、何があったっけ？
- 予定外にパーツ屋に来たけど、在庫切らしてた部品はなんだっけ？
- 店頭でお買い得品を見つけたけど、これ持ってなかったけ？

パーツはデータベース(SQLite)で管理し、GitHub Pagesでビューア(Webアプリ)をホスティングしています。

ビューアはパーツ情報の検索・表示機能のみサポートします。更新機能はありません。
パーツ情報の更新は[eparts.db](./eparts.db)を変更することで行います。リポジトリにコミットすることでビューアに反映されます。

# 環境設定

## リポジトリの設定

1. [schema.sql](./schema.sql)がデータベースのスキーマです。この形式でSQLiteでデータベースファイル`eparts.db`を生成します。
2. リポジトリをフォークまたはクローンして独自のリポジトリを作成し、`eparts.db`を上記のもので置き換えます。

 **このリポジトリの `eparts.db` ファイルは、リポジトリのオーナーのみが更新できます。これはデータベースの整合性を保つための措置です。**

## GitHub Pagesのホスティング設定

1. GitHubのリポジトリページに移動`Settings`タブをクリック
2. 左側のサイドバーから`Pages`を選択  
  `Source`セクションで、ブランチを選択
3. ルートディレクトリ「/(root)」を選択し、`Save`をクリック

これで `https://リポジトリオーナー名.github.io/ePartsDB/` でビューアにアクセスできるようになります。

## パーツの登録

`eparts.db`にパーツのデータを登録します。

[DB Browser for SQLite](https://sqlitebrowser.org)などでレコードを追加します。(煩雑なのでいずれ登録プログラムを作るかもしれません。)

- カテゴリは、gategories テーブルにレコードを追加。
- パーツ情報は、parts テーブルにレコードを追加。`category_id`を適切にセットすること。
- 在庫情報は、inventory テーブルにレコード追加。`part_id`を適切にセットすること。

テーブルの詳細は[スキーマ](./schema.sql)を参照してください。
